02A

Install directly from GitHub

If you want to inspect or modify the source, clone the repository and build the image yourself. Push it to a registry reachable by your cluster, then install the chart from the cloned repository.

BUILD AND INSTALL FROM SOURCE
git clone https://github.com/sukumarrekapalli/cnra-mvp.git
cd cnra-mvp
helm lint chart/cnra-mvp
docker buildx build --platform linux/amd64 -t sukumar9/cnra-mvp-image:source-0.2.4 --push .
helm upgrade --install cnra ./chart/cnra-mvp --namespace cnra-system --create-namespace --set image.repository=sukumar9/cnra-mvp-image --set image.tag=source-0.2.4

On an AMD64 Linux node, docker build is enough. On Apple Silicon or another ARM machine, keep --platform linux/amd64 so the image matches common cluster nodes.

CNRA runs inside your Kubernetes cluster, reads Deployment resources, applies transparent readiness rules, and serves a local dashboard with a score, evidence, findings, and remediation guidance.

02

Install the beta

CNRA requires Helm 3 and a Kubernetes context with permission to install cluster-scoped RBAC. The chart creates one read-only ServiceAccount, ClusterRole, Deployment, and Service.

  1. Confirm your cluster contextkubectl config current-context && kubectl get nodes
  2. Install directly from the public OCI chartNo Helm repository setup or registry login is required to pull the public chart.
INSTALL FROM REGISTRY
helm upgrade --install cnra oci://registry-1.docker.io/sukumar9/cnra-mvp --version 0.2.4 --namespace cnra-system --create-namespace
  1. Wait for the analyzer to become readykubectl -n cnra-system rollout status deployment/cnra-cnra-mvp
  2. Open the dashboard through a secure port-forward
OPEN THE DASHBOARD
kubectl -n cnra-system port-forward svc/cnra-cnra-mvp 8080:80

Open http://localhost:8080 in your browser. To scan one namespace only, add --set scan.namespace=my-namespace to the Helm command.

REMOVE CNRA
helm uninstall cnra --namespace cnra-system

04

Security & RBAC

CNRA does not write to your cluster. Its ClusterRole is limited to reading and watching Deployments across namespaces.

API groupapps
Resourcedeployments
Verbsget · list · watch
Write accessnone

The chart also runs the container as UID 10001, drops Linux capabilities, disables privilege escalation, uses RuntimeDefault seccomp, and mounts the root filesystem read-only.

06

API surface

The beta exposes a small HTTP surface for dashboards, probes, and integrations.

GET /healthzReturns ok when the process is alive.
GET /api/assessmentRuns a scan and returns score, dimensions, findings, services, and the rule catalog as JSON.
POST /api/assessmentRuns the same fresh scan for automation clients.
GET /api/catalogReturns the rule catalog metadata as JSON.
